## What changes in advanced setups?

| Concern               | Default CSS flow                                        | Advanced flow                                                             |
| --------------------- | ------------------------------------------------------- | ------------------------------------------------------------------------- |
| Enter and exit timing | `useToastAnimation()` reads computed CSS duration.      | Your renderer must call `markEntered()` and `markExited()`.               |
| Rendering loop        | `Toaster` and `Toaster.List` render grouped placements. | `useStore(store)` can drive a fully custom renderer.                      |
| Gestures              | `useToastDrag()` handles pointer drag out of the box.   | You can combine store actions with Framer Motion or use-gesture.          |
| Placement             | `Toaster.List` groups by `placement`.                   | You can anchor or reposition toasts as long as store state stays updated. |

## Custom animation libraries

When CSS is not the source of truth for timing, your renderer becomes responsible for lifecycle completion.

```tsx
import { useEffect } from "react";
import { AnimatePresence, motion } from "framer-motion";
import { ToastCtx, useStore, useToast } from "@headless-toast/react";
import type { ReactToastStore } from "@headless-toast/react";

function MotionToast() {
  const {
    toast: currentToast,
    dismiss,
    pauseOnHoverHandlers,
    markEntered,
    markExited,
  } = useToast();

  useEffect(() => {
    if (currentToast.status !== "entering") return;

    const timer = window.setTimeout(markEntered, 300);
    return () => window.clearTimeout(timer);
  }, [currentToast.id, currentToast.status, markEntered]);

  return (
    <motion.article
      initial={{ opacity: 0, y: -16 }}
      animate={{ opacity: 1, y: 0 }}
      exit={{ opacity: 0, y: -16 }}
      onAnimationComplete={(definition) => {
        if (definition === "exit") markExited();
      }}
      onMouseEnter={pauseOnHoverHandlers.onMouseEnter}
      onMouseLeave={pauseOnHoverHandlers.onMouseLeave}
    >
      <strong>{String(currentToast.data.title)}</strong>
      <button type="button" onClick={() => dismiss("user")}>
        Close
      </button>
    </motion.article>
  );
}

function MotionToaster({ store }: { store: ReactToastStore }) {
  const toasts = useStore(store).filter((toast) => toast.status !== "exiting");

  return (
    <AnimatePresence>
      {toasts.map((toast) => (
        <ToastCtx.Provider key={toast.id} value={{ toast, store }}>
          <MotionToast />
        </ToastCtx.Provider>
      ))}
    </AnimatePresence>
  );
}
```

## Swipe interactions

Use the built-in drag hook for pointer-driven swipe or swap in another gesture system when you need richer motion.

```tsx
import {
  useToast,
  useToastAnimation,
  useToastDrag,
} from "@headless-toast/react";

function DraggableToast() {
  const { toast: currentToast, dismiss, pauseOnHoverHandlers } = useToast();
  const drag = useToastDrag();
  const { ref, className, handlers, attributes } = useToastAnimation({
    className: "demo-toast",
    swipeDismissed: drag.swipeDismissed,
  });

  return (
    <div {...pauseOnHoverHandlers}>
      <div
        ref={ref}
        className={className}
        style={drag.style}
        {...handlers}
        {...drag.handlers}
        {...attributes}
      >
        <strong>{String(currentToast.data.title)}</strong>
        <button type="button" onClick={() => dismiss("user")}>
          Close
        </button>
      </div>
    </div>
  );
}
```

## Floating anchors

Anchored stacks are just a custom render loop driven by `useStore(store)`.

```tsx
import {
  autoUpdate,
  flip,
  offset,
  shift,
  useFloating,
} from "@floating-ui/react";

const { refs, floatingStyles } = useFloating({
  elements: { reference: buttonRef.current },
  placement: "bottom-end",
  middleware: [offset(8), flip(), shift({ padding: 16 })],
  whileElementsMounted: autoUpdate,
});

return (
  <div ref={refs.setFloating} style={floatingStyles}>
    {toasts.map((toast) => (
      <ToastCtx.Provider key={toast.id} value={{ toast, store }}>
        <FloatingToast />
      </ToastCtx.Provider>
    ))}
  </div>
);
```

## Moving toasts across placements

Placement is just toast state. If a drag target changes, update the toast and let your renderer react.

```tsx
import type { ToastPlacement } from "@headless-toast/core";

function onDropToast(id: string, nextPlacement: ToastPlacement) {
  toastStore.update(id, { placement: nextPlacement });
}
```
