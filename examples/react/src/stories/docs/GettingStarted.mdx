## Mental model

| Part                  | Responsibility                                                    | Reach for it when                                                    |
| --------------------- | ----------------------------------------------------------------- | -------------------------------------------------------------------- |
| `createToast()`       | Creates a store and imperative API.                               | You want an isolated toast system for one app shell or feature area. |
| `Toaster`             | Creates the live region and portal or inline mounting behavior.   | You need a root place where toast groups render.                     |
| `Toaster.List`        | Groups toasts by placement and exposes placement data attributes. | You want to style stack position and alignment.                      |
| `useToast()`          | Gives the current toast state and actions.                        | You are rendering the toast item itself.                             |
| `useToastAnimation()` | Reads DOM timing and advances enter and exit phases.              | Your toast uses CSS transitions or animations.                       |
| `AnimationWrapper`    | Convenience wrapper around `useToastAnimation()`.                 | You want less manual wiring for CSS-driven toasts.                   |

## Runtime flow

1. Create a store with `createToast()`.
2. Render `Toaster` once and place `Toaster.List` inside it.
3. Build a toast item component with `useToast()`.
4. Add `useToastAnimation()` so the store can wait for real DOM motion.
5. Trigger notifications through `toastStore.success()`, `toastStore.error()`, or `toastStore.promise()`.

## Smallest working setup

```tsx
import {
  createToast,
  Toaster,
  useToast,
  useToastAnimation,
} from "@headless-toast/react";
import "./toast.css";

const { toast: toastStore } = createToast<{
  title: string;
  body?: string;
}>({
  maxToasts: 3,
  defaults: {
    duration: 4000,
    placement: "top-right",
    pauseOnHover: true,
    dismissible: true,
  },
});

function AppToast() {
  const { toast: currentToast, dismiss, pauseOnHoverHandlers } = useToast();
  const { ref, className, handlers, attributes } = useToastAnimation({
    className: "app-toast",
  });

  return (
    <article
      ref={ref}
      className={className}
      {...handlers}
      {...pauseOnHoverHandlers}
      {...attributes}
    >
      <div>
        <strong>{String(currentToast.data.title)}</strong>
        {currentToast.data.body ? (
          <p>{String(currentToast.data.body)}</p>
        ) : null}
      </div>

      <button type="button" onClick={() => dismiss("user")}>
        Close
      </button>
    </article>
  );
}

export function App() {
  return (
    <>
      <button
        type="button"
        onClick={() =>
          toastStore.success({
            title: "Saved",
            body: "Profile updated successfully.",
          })
        }
      >
        Show toast
      </button>

      <Toaster store={toastStore} className="app-toast-region">
        <Toaster.List className="app-toast-list">
          <AppToast />
        </Toaster.List>
      </Toaster>
    </>
  );
}
```

## Trigger built-in toast types

```tsx
toastStore.success({ title: "Saved", body: "Everything worked." });
toastStore.error({ title: "Failed", body: "Please try again." });
toastStore.warning({ title: "Careful", body: "Double-check this change." });
toastStore.info({ title: "Heads up", body: "Background sync is running." });
toastStore.loading({ title: "Uploading", body: "Please wait..." });
```

## Set placement and timing

```tsx
toastStore.info(
  {
    title: "Bottom center",
    body: "This toast renders in a custom placement.",
  },
  {
    placement: "bottom-center",
    duration: 5000,
    pauseOnHover: true,
    dismissible: true,
  },
);
```

## Good defaults

```tsx
const { toast: toastStore } = createToast({
  maxToasts: 3,
  defaults: {
    duration: 4000,
    placement: "top-right",
    pauseOnHover: true,
    dismissible: true,
  },
});
```

- `maxToasts: 3` keeps stacks readable.
- `duration: 4000` fits short titles and short body copy.
- `pauseOnHover: true` makes auto-close less aggressive.
- `dismissible: true` gives users an immediate manual escape hatch.

Use `duration: 0` for action-required toasts, and enable `progress: true` when remaining lifetime should be visible.

## What each layer should style

| Layer            | Your styling job                                                          | Helpful selectors                                                                          |
| ---------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| `Toaster` region | Position the global overlay or inline region and set pointer-event rules. | `[data-toast-region]`, `[data-inline="true"]`                                              |
| `Toaster.List`   | Define width, spacing, alignment, and placement-specific layout.          | `[data-toast-list]`, `[data-placement]`, `[data-stack-expanded]`                           |
| Toast item       | Define surface, copy hierarchy, icons, progress UI, and motion.           | `[data-toast]`, `[data-toast-type]`, `[data-toast-status]`, `[data-toast-swipe-dismissed]` |

## Styling guidance

- Keep the toast width around `18rem` to `24rem` and cap mobile width with `calc(100vw - 16px)`.
- Let the region ignore pointer events and turn them back on at the toast item level.
- Keep enter motion around `220ms` to `280ms` and exit motion around `160ms` to `220ms`.
- Move a toast only a little during animation; `8px` to `14px` is usually enough.
- Prefer one accent per toast type over five different layouts.

```css
.app-toast-region {
  position: fixed;
  inset: 0;
  pointer-events: none;
  z-index: 9999;
}

.app-toast-list {
  position: fixed;
  top: 0;
  right: 0;
  display: flex;
  flex-direction: column;
  gap: 12px;
  width: min(24rem, calc(100vw - 24px));
  max-width: calc(100vw - 24px);
  padding: 16px;
}

.app-toast {
  pointer-events: auto;
  display: flex;
  justify-content: space-between;
  gap: 12px;
  padding: 14px 16px;
  border: 1px solid rgba(15, 23, 42, 0.12);
  border-radius: 18px;
  background: rgba(255, 255, 255, 0.96);
  box-shadow: 0 18px 40px rgba(15, 23, 42, 0.14);
}

.app-toast[data-toast-status="entering"] {
  animation: toast-enter 240ms cubic-bezier(0.16, 1, 0.3, 1);
}

.app-toast[data-toast-status="exiting"][data-toast-swipe-dismissed="false"] {
  animation: toast-exit 180ms ease-in forwards;
}

@keyframes toast-enter {
  from {
    opacity: 0;
    transform: translateY(-10px) scale(0.98);
  }

  to {
    opacity: 1;
    transform: translateY(0) scale(1);
  }
}

@keyframes toast-exit {
  from {
    opacity: 1;
    transform: translateY(0) scale(1);
  }

  to {
    opacity: 0;
    transform: translateY(-8px) scale(0.98);
  }
}
```
