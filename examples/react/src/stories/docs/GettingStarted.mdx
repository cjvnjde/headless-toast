# Getting started

This Storybook is organized around small, focused stories so you can inspect one behavior at a time.

## Usage flow

1. Create a toast store with `createToast()`.
2. Mount a `Toaster` with your own toast component.
3. Trigger notifications through the returned `toast` api.
4. Add optional behavior like progress, pause-on-hover, or placement overrides.

## How to create the smallest working setup

```tsx
import {
  createToast,
  Toaster,
  useToast,
  useToastAnimation,
} from "@headless-toast/react";

const { store, toast } = createToast();

function AppToast() {
  const { toast: item, dismiss, pauseOnHoverHandlers } = useToast();
  const { ref, className, handlers, attributes } = useToastAnimation({
    className: "demo-toast",
  });

  return (
    <div
      ref={ref}
      className={className}
      {...handlers}
      {...pauseOnHoverHandlers}
      {...attributes}
    >
      <strong>{String(item.data.title)}</strong>
      <p>{String(item.data.body)}</p>
      <button onClick={() => dismiss("user")}>Close</button>
    </div>
  );
}

export function App() {
  return (
    <>
      <button
        onClick={() =>
          toast.success({
            title: "Saved",
            body: "Profile updated successfully.",
          })
        }
      >
        Show toast
      </button>

      <Toaster store={store} component={AppToast} />
    </>
  );
}
```

## How to trigger different toast types

```tsx
toast.success({ title: "Saved", body: "Everything worked." });
toast.error({ title: "Failed", body: "Please try again." });
toast.warning({ title: "Careful", body: "Double-check this change." });
toast.info({ title: "Heads up", body: "Background sync is running." });
toast.loading({ title: "Uploading", body: "Please wait..." });
```

## How to set placement and timing

```tsx
toast.info(
  {
    title: "Bottom center",
    body: "This toast renders in a custom placement.",
  },
  {
    placement: "bottom-center",
    duration: 5000,
    pauseOnHover: true,
    dismissible: true,
  },
);
```

## Good defaults to start with

If you do not have a strong product-specific opinion yet, these defaults tend to feel solid in most apps:

```tsx
const { store, toast } = createToast({
  maxToasts: 3,
  defaults: {
    duration: 4000,
    placement: "top-right",
    pauseOnHover: true,
    dismissible: true,
  },
});
```

- `duration: 4000` gives people enough time to scan a short title and body.
- `placement: "top-right"` stays out of the way and matches common desktop app patterns.
- `pauseOnHover: true` makes timed toasts feel less aggressive.
- `dismissible: true` gives users an immediate escape hatch.
- `maxToasts: 3` keeps the stack readable instead of turning into a wall of notifications.

Use `duration: 0` for toasts that need an explicit user action, and add `progress: true` when the remaining lifetime should be visible.

## Styling guidance

`@headless-toast/react` is fully headless. The toast store controls state and timing, but your component still needs real CSS for layout, colors, and enter/exit motion.

Good visual defaults:

- Keep the toast width around `18rem` to `24rem`.
- Use `12px 16px` padding and a `12px` to `16px` radius.
- Start with a neutral surface, strong title, softer body copy, and one accent color per toast type.
- Use subtle motion: about `220ms` to `280ms` for enter and `160ms` to `200ms` for exit.
- Move only a little during animation, usually `8px` to `14px` plus a slight scale change.

```css
.demo-toast {
  min-width: 18rem;
  max-width: 24rem;
  display: flex;
  align-items: flex-start;
  gap: 0.75rem;
  padding: 12px 16px;
  border: 1px solid #d0d7de;
  border-radius: 14px;
  background: #ffffff;
  color: #0f172a;
  box-shadow: 0 18px 40px rgba(15, 23, 42, 0.14);
}

.demo-toast[data-toast-status="entering"] {
  animation: toast-enter 240ms cubic-bezier(0.16, 1, 0.3, 1);
}

.demo-toast[data-toast-status="exiting"][data-toast-swipe-dismissed="false"] {
  animation: toast-exit 180ms ease-in forwards;
}

.demo-toast[data-toast-status="exiting"][data-toast-swipe-dismissed="true"] {
  animation: toast-swipe-exit 180ms ease-in forwards;
}

@keyframes toast-enter {
  from {
    opacity: 0;
    transform: translateY(-10px) scale(0.98);
  }

  to {
    opacity: 1;
    transform: translateY(0) scale(1);
  }
}

@keyframes toast-exit {
  from {
    opacity: 1;
    transform: translateY(0) scale(1);
  }

  to {
    opacity: 0;
    transform: translateY(-8px) scale(0.98);
  }
}

@keyframes toast-swipe-exit {
  from {
    opacity: 1;
  }

  to {
    opacity: 0;
  }
}
```

The important part is that your rendered element has a real enter and exit animation or transition. `useToastAnimation()` reads that duration from the DOM so the store knows when to move between phases.

## Tailwind defaults

If you are styling with Tailwind, this is a good baseline for a neutral app toast:

```tsx
function AppToast() {
  const { toast: item, dismiss, pauseOnHoverHandlers } = useToast();
  const { ref, className, handlers, attributes } = useToastAnimation({
    className:
      "toast-shell relative flex min-w-72 max-w-sm items-start gap-3 overflow-hidden rounded-2xl border border-slate-200 bg-white p-4 text-slate-900 shadow-lg",
  });

  return (
    <div
      ref={ref}
      className={className}
      {...handlers}
      {...pauseOnHoverHandlers}
      {...attributes}
    >
      <div className="mt-0.5 flex size-9 shrink-0 items-center justify-center rounded-xl bg-emerald-500 text-sm font-semibold text-white">
        OK
      </div>

      <div className="min-w-0 flex-1 pr-8">
        <strong className="block text-sm font-semibold tracking-tight text-slate-900">
          {String(item.data.title)}
        </strong>
        {item.data.body ? (
          <p className="mt-1 text-sm leading-5 text-slate-600">
            {String(item.data.body)}
          </p>
        ) : null}
      </div>

      <button
        className="absolute right-3 top-3 inline-flex size-8 items-center justify-center rounded-full text-slate-400 hover:bg-slate-100 hover:text-slate-700"
        onClick={() => dismiss("user")}
      >
        x
      </button>
    </div>
  );
}
```

Good Tailwind defaults:

- `min-w-72 max-w-sm` for a readable width without overgrowing small screens.
- `rounded-2xl border border-slate-200 bg-white shadow-lg` for a clean neutral surface.
- `gap-3 p-4` for spacing that feels roomy without wasting space.
- `text-sm`, `font-semibold`, and `text-slate-600` for a clear title/body hierarchy.
- A small accent block or icon per toast type instead of recoloring the entire surface.

Tailwind still needs explicit animation rules because the adapter reads the computed duration from the DOM:

```css
.toast-shell[data-toast-status="entering"] {
  animation: toast-enter 240ms cubic-bezier(0.16, 1, 0.3, 1);
}

.toast-shell[data-toast-status="exiting"][data-toast-swipe-dismissed="false"] {
  animation: toast-exit 180ms ease-in forwards;
}

.toast-shell[data-toast-status="exiting"][data-toast-swipe-dismissed="true"] {
  animation: toast-swipe-exit 180ms ease-in forwards;
}
```

## How to enforce a maximum number of visible toasts

```tsx
const { store, toast } = createToast({ maxToasts: 3 });

toast.info({ title: "One" });
toast.info({ title: "Two" });
toast.info({ title: "Three" });
toast.info({ title: "Four" });
// The oldest visible toast is dismissed first.
```
