## Which primitive should you reach for?

| Goal                                         | Primary primitive                                        | Why                                                              |
| -------------------------------------------- | -------------------------------------------------------- | ---------------------------------------------------------------- |
| Keep feedback inside a panel or form         | `containerId` + `inline` toaster                         | One store can drive local and global toasts without duplication. |
| Build counters, dashboards, or custom stacks | `useStore(store)`                                        | You get the raw toast array and render anything from it.         |
| Fully own markup and styling                 | `useToast()`                                             | You control every element and interaction of one toast item.     |
| Keep the system utility-first                | `useToastAnimation()` + Tailwind                         | Lifecycle stays stable while styling changes freely.             |
| Replace or refresh an existing toast         | Stable `id` + `toastStore.update()` or repeated triggers | A stable id turns a new trigger into an in-place update.         |

## Scoped containers

Use `containerId` when notifications should stay inside a sidebar, drawer, panel, or form.

```tsx
import {
  createToast,
  Toaster,
  useToast,
  useToastAnimation,
} from "@headless-toast/react";

const { toast: toastStore } = createToast();

function InlineToast() {
  const { toast: currentToast, dismiss } = useToast();
  const { ref, className, handlers, attributes } = useToastAnimation({
    className: "inline-toast",
  });

  return (
    <div ref={ref} className={className} {...handlers} {...attributes}>
      <div>
        <strong>{String(currentToast.data.title)}</strong>
        <span>{String(currentToast.data.body)}</span>
      </div>
      <button type="button" onClick={() => dismiss("user")}>
        Close
      </button>
    </div>
  );
}

function SidebarFeedback() {
  return (
    <Toaster store={toastStore} containerId="sidebar" inline>
      <Toaster.List className="sidebar-toast-list">
        <InlineToast />
      </Toaster.List>
    </Toaster>
  );
}

toastStore.success(
  { title: "Saved", body: "Sidebar settings updated." },
  { containerId: "sidebar", duration: 3000 },
);
```

## Reactive inspection

Use `useStore(store)` when you need counters, devtools, analytics overlays, or custom render loops.

```tsx
import { useStore } from "@headless-toast/react";
import type { ReactToastStore } from "@headless-toast/react";

function ToastDebugPanel({ store }: { store: ReactToastStore }) {
  const toasts = useStore(store);

  return (
    <ul>
      {toasts.map((toast) => (
        <li key={toast.id}>
          {toast.type} - {toast.status} - {toast.remaining}ms remaining
        </li>
      ))}
    </ul>
  );
}
```

## Custom rendering

Use `useToast()` when you want design-system markup instead of a demo shell.

```tsx
import { useToast, useToastAnimation } from "@headless-toast/react";

function BrandToast() {
  const { toast: currentToast, dismiss } = useToast();
  const { ref, className, handlers, attributes } = useToastAnimation({
    className: "brand-toast",
  });

  return (
    <article ref={ref} className={className} {...handlers} {...attributes}>
      <div>
        <strong>{String(currentToast.data.title)}</strong>
        {currentToast.data.body ? (
          <p>{String(currentToast.data.body)}</p>
        ) : null}
      </div>
      <button type="button" onClick={() => dismiss("user")}>
        Dismiss
      </button>
    </article>
  );
}
```

## Tailwind v4 styling

Tailwind changes only the presentation layer. The store, toaster, and hooks stay exactly the same.

```tsx
import {
  createToast,
  Toaster,
  useToast,
  useToastAnimation,
} from "@headless-toast/react";
import "./tailwind.css";

const { toast: toastStore } = createToast();

function TailwindToast() {
  const { toast: currentToast, dismiss, pauseOnHoverHandlers } = useToast();
  const { ref, className, handlers, attributes } = useToastAnimation({
    className: "toast-shell",
  });

  return (
    <div
      ref={ref}
      className={className}
      {...handlers}
      {...pauseOnHoverHandlers}
      {...attributes}
    >
      <div className="min-w-0 flex-1 pr-8">
        <strong className="block text-sm font-semibold text-[var(--color-toast-ink)]">
          {String(currentToast.data.title)}
        </strong>
        {currentToast.data.body ? (
          <p className="mt-1 text-sm text-[var(--color-toast-muted)]">
            {String(currentToast.data.body)}
          </p>
        ) : null}
      </div>

      <button
        className="toast-close-btn"
        type="button"
        onClick={() => dismiss("user")}
      >
        x
      </button>
    </div>
  );
}

<Toaster
  store={toastStore}
  className="pointer-events-none fixed inset-0 z-[9999]"
>
  <Toaster.List className="pointer-events-none fixed right-0 top-0 flex w-[min(26rem,calc(100vw-24px))] max-w-[calc(100vw-24px)] flex-col items-end gap-3 p-6">
    <TailwindToast />
  </Toaster.List>
</Toaster>;
```

```css
@import "tailwindcss";

.toast-shell[data-toast-status="entering"] {
  animation: toast-tailwind-enter 300ms cubic-bezier(0.16, 1, 0.3, 1);
}

.toast-shell[data-toast-status="exiting"][data-toast-swipe-dismissed="false"] {
  animation: toast-tailwind-exit 240ms ease-in forwards;
}
```

## Updating an existing toast by id

Stable ids are the easiest way to dedupe or turn loading into success and error states.

```tsx
toastStore.loading(
  { title: "Uploading", body: "Starting upload..." },
  { id: "file-upload", duration: 0 },
);

toastStore.success(
  { title: "Uploaded", body: "All files are stored." },
  { id: "file-upload", duration: 3000 },
);
```
